Binary search trees are fundamental data structures in computer science that enable efficient searching, insertion, and deletion operations. Unlike linear data structures such as arrays or linked lists, binary search trees organize elements hierarchically. Each node contains a value and pointers to at most two children: left and right. The defining property is that all values in the left subtree are smaller than the node's value, while all values in the right subtree are larger.

This ordering property enables logarithmic time complexity for basic operations in balanced trees. However, when a binary search tree becomes unbalanced, performance degrades to linear time. Self-balancing variants like AVL trees and red-black trees maintain balance through rotations during insertions and deletions. AVL trees enforce strict balance criteria, ensuring the height difference between subtrees never exceeds one. Red-black trees use color properties and relaxed balancing rules for faster insertions.

Applications include implementing associative arrays, priority queues, and maintaining sorted collections. Database indexing systems frequently employ B-trees, which generalize binary search trees to allow more than two children per node. This reduces tree height and minimizes disk accesses in storage systems. Understanding these structures is essential for algorithm design and optimization in software engineering.
