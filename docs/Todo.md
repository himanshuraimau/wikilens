## 🔴 Backend — Critical / Core Missing

### Wikitext Markup Stripper
The XML parser feeds raw Wikitext directly into the tokenizer with no cleanup.
This degrades index quality significantly.

- [ ] Strip templates: `{{Infobox person | ...}}`
- [ ] Convert internal wiki links: `[[Alan Turing|Turing]]` → keep display text
- [ ] Remove external links: `[http://example.com Example]` → keep `Example`
- [ ] Remove heading markers: `== Early life ==` → keep `Early life`
- [ ] Remove bold/italic markers: `'''bold'''` and `''italic''`
- [ ] Remove reference tags: `<ref>...</ref>` (including content)
- [ ] Remove HTML comments: `<!-- ... -->`
- [ ] Remove tables: `{| ... |}` (skip for v1 if too complex)
- [ ] Create `src/wikitext_stripper.cpp` + `include/wikitext_stripper.h`
- [ ] Wire it into the ingestion pipeline in `main.cpp` between the XML parser callback and the tokenizer

### Link Graph Extraction & PageRank
- [ ] While stripping Wikitext, extract every `[[Internal Link]]` and record edge `(source_title → target_title)`
- [ ] Save link graph as a plain-text edge list file (e.g., `data/link_graph.txt`)
- [ ] Implement offline PageRank computation (iterative power method, ~20–50 iterations)
- [ ] Create `src/pagerank.cpp` + `include/pagerank.h`
- [ ] Store per-document PageRank scores and make them accessible to the query engine
- [ ] Combine BM25 score + PageRank into a final blended score in `QueryEngine::search()`

### Index Persistence (Disk Serialization)
Currently the index is rebuilt from scratch every startup, which is very slow at scale.

- [ ] Implement `InvertedIndex::saveToDisk(const std::string& path)` in `inverted_index.cpp`
- [ ] Implement `InvertedIndex::loadFromDisk(const std::string& path)` in `inverted_index.cpp`
- [ ] Use a binary format: term dictionary header + posting list data
- [ ] On startup in `main.cpp`: check if a saved index exists → load it; otherwise build and save
- [ ] Also persist `DocumentStore` to disk and reload it

### Snippet / Excerpt Extraction
- [ ] Store a short plain-text excerpt (first ~300 chars of stripped body) per document in `DocumentStore`
- [ ] Update `Document` struct in `include/document_store.h` to include `std::string snippet`
- [ ] Return `snippet` field in the `/search` JSON response in `main.cpp`

### BM25 `avgDocLength()` Implementation
- [ ] Implement `InvertedIndex::avgDocLength()` in `inverted_index.cpp` — currently declared but likely a stub
- [ ] Compute it as `total_token_count / total_docs_` and cache it after index build

### Wikipedia URL Construction
- [ ] Construct proper Wikipedia URLs in the ingestion pipeline: `https://en.wikipedia.org/wiki/{URL-encoded title}`
- [ ] Store this URL in `DocumentStore` and return it in the search API response
